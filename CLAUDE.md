## Файловая структура

```
Презентация/
├── index.html              ← короткий сборщик: fetch+DOMParser, манифест SLIDES, печать
├── CLAUDE.md               ← этот файл
├── PLAN.md                 ← контекст проекта, цифры, прогресс
├── styles/
│   ├── tokens.css          ← :root переменные (палитра, шрифты, геометрия), reset, html/body
│   ├── base.css            ← .deck, .slide, шапка, футер, .nav-hint, @page, @media print
│   └── slide-N.css         ← всё специфичное для слайда N
├── slides/
│   └── slide-N.html        ← самодостаточная HTML-страница с одним <section class="slide">
└── assets/
    └── logos/...
```

## Как работают слайды

- **Каждый `slides/slide-N.html`** — полноценная HTML-страница, открывается в браузере напрямую (для дебага одного слайда). Подключает `../styles/tokens.css`, `../styles/base.css`, `../styles/slide-N.css`. Содержит один `<section class="slide" id="slide-N">`.
- **`index.html`** — точка входа дека. Подключает общие стили + все `slide-N.css`. JS-сборщик: `fetch` каждого `slides/slide-N.html`, `DOMParser` извлекает `<section>`, переписывает `../assets/*` → `assets/*`, вставляет в `.deck`. До завершения сборки `beforeprint` блокирует печать.

## Как добавить новый слайд

1. Создать `slides/slide-N.html` (скопировать структуру с slide-1).
2. Создать `styles/slide-N.css`.
3. В `index.html` добавить путь в манифест `SLIDES` и `<link>` на `slide-N.css`.
4. Обновить нумерацию страниц в футере (`NN / 15`).
