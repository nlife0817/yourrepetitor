# Презентация «Твой репетитор» × «Создай НАШЕ 2026»

Компактные правила для сессий Claude. Длинные конвенции — в `.claude/skills/deck/SKILL.md` (подгружается прогрессивно). Эстетика — в [DESIGN.md](./DESIGN.md). Референс по архитектуре — в [presentations-claude-ru.md](./presentations-claude-ru.md).

## Стек
**Slidev** (Markdown + Vue 3 + UnoCSS). Слайд = `pages/NN-name.md` ≤200 токенов. Точка входа — `slides.md` (только frontmatter + `src:` импорты). Переиспользуемая разметка — `components/*.vue`, обёртки слайдов — `layouts/*.vue`, токены — `styles/tokens.css`. Контент (числа/тезисы) — в `data/content.yml`.

## Команды
- `pnpm dev` — локальный сервер с HMR
- `pnpm export` — PDF (для сдачи)
- `pnpm build` — статический SPA (для публикации)

## Размер и формат
- Slidev canvas: **1920 × 1080**, соотношение 16:9 (`canvasWidth: 1920` в `slides.md`)
- Шрифты: **Onest** (UI, веса 300/400/500/600/700), **JetBrains Mono** (числа/email/даты/цены, веса 400/500/600). Подключение в `index.html` или через `fonts:` в frontmatter.
- Палитра — `styles/tokens.css` (CSS-переменные). Главные: `--bg #f6f5f1`, `--accent #e8734a`, `--text-primary #1a1a1f`, `--text-secondary #6b6d7b`, `--orange-text #c2603a`, `--indigo #6366f1`. Полный список — в `tokens.css`.

## Workflow с автором (КРИТИЧНО)
Для каждого слайда: **сначала текст словами в чате → согласование → только потом код**. «Поехали» = «начинаем обсуждение слайда», не «пиши код».

## Правила правок
- Правь `pages/NN-*.md`, **не** `slides.md`.
- Дизайн-токены меняй в `styles/tokens.css` — никогда не хардкодь цвет/шрифт/отступ в слайде или компоненте.
- Повторяющийся визуальный паттерн (мокап, статистика, цитата, чарт) → новый компонент в `components/`, не копи-паста.
- Бинарные мокапы и сложные SVG — в `public/`, подключай как `<img src="/mockups/...">`. Не генерируй пиксельный CSS заново на каждый ход.
- Эстетика — см. `DESIGN.md`. Не используй generic AI-стиль (Inter/Roboto, фиолетовые градиенты на белом, и т.п.).

## Делегирование и контекст
- Сложную верстку слайда / компонента — через агента `frontend-developer` (Agent tool).
- Большие выводы (анализ конкурентов, сканы документов) — сабагент пишет в файл, возвращает путь + summary, **не** дамп в чат.
- На границах задач — `/compact` с инструкцией сохранить: модифицированные файлы, символы, открытые TODO, следующий шаг.

## Запреты
- Не использовать эмодзи в коде/контенте, если автор не попросил.
- Не дублировать токены (цвет/шрифт/отступ) в компонентах — только переменные из `tokens.css`.
- Не редактировать `CLAUDE.md`, `.claude/skills/`, MCP-настройки в активной сессии — это ломает prompt cache. Правки оформлять как TODO в `.claude/HANDOFF.md` и применять в новой сессии.
- Не комментировать очевидное в коде.

## Старт новой сессии
Прочесть в порядке: **этот файл → `.claude/HANDOFF.md` → `DESIGN.md` (при дизайн-задачах)**. Затем спросить автора, с какого слайда продолжаем. Без явного «поехали» работу не начинать.
