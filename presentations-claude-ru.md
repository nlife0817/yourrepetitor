## Ключевые идеи изменения структуры проекта 

- **Перейти от «один HTML+CSS на слайд, всё руками» к code-first фреймворку с прогрессивным раскрытием.** Самый сильный шаг — взять **Slidev** (Markdown + Vue-компоненты + UnoCSS, исходник одного слайда ≈ 20–80 токенов) как уровень авторинга, разбить контент по `pages/*.md` и подключать через `src:` во frontmatter, а весь переиспользуемый дизайн вынести в `components/`, `layouts/`, `styles/` и в общий файл с CSS-переменными. Официальный PPTX-skill самой Anthropic (`anthropics/skills`, `pptx/SKILL.md`) делает ровно то же: *«Создай отдельный HTML-файл на каждый слайд... В общем .css-файле переопределяй CSS-переменные для цветов, типографики и отступов»*.

- **Обернуть процесс в примитивы Claude Code, специально созданные ради экономии контекста:** компактный `CLAUDE.md`, проектный Skill в `.claude/skills/deck/` с прогрессивным раскрытием (по best practices Anthropic: *«держи тело SKILL.md в пределах 500 строк для оптимальной производительности»*), Plan Mode для правок, sub-agents для визуального QA, и поддержка prompt caching по принципу *«статика впереди, динамика в конце»* (Тарик Шихипар, «Lessons from building Claude Code: Prompt caching is everything», 30 апреля 2026, claude.com/blog).

- **Не путай «экономный код» с «экономным дизайном».** Используй официальный skill `frontend-design` от Anthropic плюс `DISTILLED_AESTHETICS_PROMPT` из Frontend Aesthetics Cookbook, чтобы типографика, цвет и движение оставались смелыми. Сложные мокапы устройств, градиенты и плотную графику растеризуй один раз в PNG/SVG и подключай ссылкой — никогда не генерируй пиксельный CSS заново на каждой итерации. Экспорт в PDF и SPA — одной командой `slidev export`.

## Ключевые выводы

1. **Твоя текущая архитектура — худший случай для контекста.** Один HTML + один CSS на каждый «креативный» слайд раздувается до ~10K токенов, потому что каждый слайд повторяет обёртки, шрифты, базовые стили, layout-примитивы и значения цветов. По инженерному блогу Anthropic «Effective context engineering for AI agents» (29 сентября 2025): *«Исследования needle-in-a-haystack обнаружили феномен context rot: с ростом числа токенов в окне модель хуже извлекает информацию»*. Механика прямо названа в той же статье: *«Это даёт n² парных связей для n токенов»*. (Anthropic подчёркивает, что термин «context rot» взят у внешнего исследования Chroma.) Сокращение размера исходника одного слайда — это одновременно экономия и качество.

2. **Slidev — наилучший фит под AI-генерируемые дизайнерские деки.** Markdown-исходник, Vue 3 компоненты, UnoCSS (utility-first → понятные LLM токены), экспорт в PDF/PPTX/SPA одной командой, мульти-вход через `src: ./pages/*.md`, кастомные `layouts/` и `components/` для переиспользования. Reveal.js, Marp, Spectacle и Astro+Reveal жизнеспособны, но у каждого свои компромиссы (см. сравнение).

3. **Официальный PPTX-skill Anthropic подтверждает паттерн «общий CSS + один HTML на слайд».** Из `anthropics/skills/skills/pptx/SKILL.md`: *«Создай HTML-файл на каждый слайд с правильными размерами (например, 720pt × 405pt для 16:9)»* и *«В общем .css-файле переопределяй CSS-переменные на каждом слайде для цветов, типографики, отступов. НЕ создавай классы в этом файле»*. Именно эта дисциплина держит токенную стоимость слайда плоской.

4. **Skills + sub-agents + prompt caching — канонический стек экономии токенов.** Метаданные skill'а на старте загружаются только как имя + описание. Из «Equipping agents for the real world with Agent Skills» от Anthropic: *«эти метаданные — первый уровень прогрессивного раскрытия... они дают ровно столько информации, чтобы Claude знал, когда применить каждый skill, не загружая его целиком в контекст»*. Тело подгружается только при срабатывании; reference-файлы — только при чтении. По прайсингу Anthropic: *«Попадание в кеш стоит 10% от обычной цены input-токенов»*. А по доке Claude Code Skills: *«Claude Code заново прикрепляет последний вызов каждого skill после summary, сохраняя первые 5000 токенов каждого. Re-attached skills делят общий бюджет в 25 000 токенов»*.

5. **Существует готовый дизайн-промпт против AI-слопа в слайдах.** Frontend Aesthetics Cookbook (`platform.claude.com/cookbook/coding-prompting-for-frontend-aesthetics`, Притви Раджасекаран, Anthropic, 21 октября 2025) даёт `DISTILLED_AESTHETICS_PROMPT` с явными запретами на Inter/Roboto/Arial, «фиолетовые градиенты на белом» и предсказуемые сходимости к Space Grotesk. Положи блок дословно в `DESIGN.md` или в SKILL.md деки.

## Подробности

### 1. Сравнение фреймворков для AI-генерируемых дизайнерских деков

| Фреймворк | Компактность исходника на слайд | PDF-экспорт | Дизайн-потолок | Удобство AI-редактирования |
|---|---|---|---|---|
| **Slidev** (Vue 3 + Vite + UnoCSS) | **Лучшая.** Слайд может быть 3–10 строк Markdown, разрешающихся в Vue layout. `src: ./pages/foo.md` режет деку на отдельные файлы по ~20–100 токенов. UnoCSS-классы короткие и предсказуемые для LLM. | `slidev export` собирает PDF, PPTX, PNG, SPA одной командой (`sli.dev/guide/exporting`). | Очень высокий — Vue-компоненты, WebGL, SVG, Motion, Monaco, кастомные layouts. Антоний Фу пишет: *«всё, что можно сделать в веб-приложении, можно применить к слайдам»*. | Отличное. Frontmatter маленький, декларативный, дружелюбный к diff'ам. Кастомные `layouts/` + `components/` дают Claude стабильный контракт вместо регенерации CSS. |

**Вердикт:** под твой кейс (12–20 слайдов, мокапы устройств, графики, креативная типографика, выходы PDF + веб) **Slidev — основной инструмент**, а PPTX-skill Anthropic — запасной путь, когда нужен именно `.pptx`.

### 2. Рекомендуемая архитектура проекта

```
my-deck/
├── CLAUDE.md                 # Компактный: build-команды, токены палитры, do/don't, ссылки на skills
├── AGENTS.md                 # (Опционально) зеркало CLAUDE.md для других агентов
├── DESIGN.md                 # Зафиксированное эстетическое направление + DISTILLED_AESTHETICS_PROMPT дословно
├── slides.md                 # Точка входа: только frontmatter + `src:` импорты
├── pages/                    # Один файл на слайд, 20–200 токенов каждый
│   ├── 01-cover.md
│   ├── 02-problem.md
│   └── …
├── components/               # Переиспользуемые Vue-компоненты: <DeviceMock />, <Stat />, <Quote />
│   ├── DeviceMock.vue
│   ├── Stat.vue
│   └── BarChart.vue
├── layouts/                  # Кастомные layouts: hero, two-col-60-40, fullbleed
├── styles/
│   ├── tokens.css            # Только CSS-переменные (--color-bg, --font-display, --space-*)
│   └── index.ts              # Импортирует tokens + минимальные глобалы
├── public/
│   ├── mockups/              # Заранее отрендеренные PNG/WebP мокапы устройств
│   └── illustrations/        # SVG (дёшевы по токенам, идеально масштабируются)
├── data/
│   └── content.yml           # Чистый контент, отделённый от шаблонов (заголовки, буллеты, цифры)
├── .claude/
│   ├── skills/
│   │   └── deck/
│   │       ├── SKILL.md          # ≤500 строк по best practice Anthropic
│   │       ├── component_patterns.md
│   │       └── pdf_export.md
│   └── settings.json         # Hooks, разрешённые команды
└── package.json
```

**Почему это экономит токены (что меняется на слайд):**

- Слайд становится таким: `--- src: ./pages/05-stats.md --- # Q3 results <Stat label="MRR" value="$1.2M" delta="+18%"/>`. ~30 токенов вместо текущих ~10K.
- Мокапы, графики, декоративные SVG живут как бинарные/SVG-ассеты в `public/`. Claude читает `<img src="/mockups/iphone-feed.png" />` (~10 токенов), а не пиксельную сетку.
- Все дизайн-решения — в `styles/tokens.css` + Vue-компоненты. «Сделай бренд-цвет теплее» = правка одной CSS-переменной, а не 20 слайдов.
- Правка слайда 5 читает только `pages/05-stats.md` плюс нужный компонент — а не всю деку.

### 3. Конкретные техники Claude Code

**a) `CLAUDE.md` — держи компактным.** По доке Best Practices, CLAUDE.md загружается в каждое взаимодействие; раздутость оплачивается каждый ход. Ограничься: build/test-командами (`pnpm dev`, `slidev export --format pdf`), палитрой и шрифтами, голосом деки, правилом «всегда правь `pages/*.md`, а не `slides.md`», и одной строкой ссылок на `DESIGN.md` и `.claude/skills/deck/`.

**b) Проектный skill `.claude/skills/deck/SKILL.md` с прогрессивным раскрытием.** Описание + триггер. Тело перечисляет конвенции авторинга, доступные Vue-компоненты с пропсами, экспорт-команды, и ссылки на длинные reference-файлы (`component_patterns.md`, `pdf_export.md`), которые Claude читает только по запросу. Best practice Anthropic: *«держи тело SKILL.md в пределах 500 строк»*. На старте метаданные skill'а стоят только токены имени + описания; тело подгружается при срабатывании.

**c) Sub-agents для визуального QA, генерации мокапов и пред-поиска.** PPTX SKILL.md прямо говорит: *«⚠️ ИСПОЛЬЗУЙ САБАГЕНТОВ — даже для 2–3 слайдов. Ты залип в коде и видишь то, что ожидаешь увидеть, а не то, что там есть. У сабагентов свежий взгляд»*. Подключи `slide-reviewer` сабагента, который через Playwright делает скриншот каждого слайда и возвращает summary проблем — основной поток остаётся чистым.

**d) Plan Mode + Explore Mode для правок.** По Best Practices Anthropic: войди в Plan Mode, попроси Claude перечислить, какие `pages/*.md` и компоненты он тронет, сохрани план, сделай `/clear` и выполни уже на свежем контексте.

**e) Дисциплина prompt caching.** Из поста Тарика Шихипара про Claude Code (30 апреля 2026): *«В инженерии часто говорят 'cache rules everything around me' — для агентов это правило тоже работает»*, и *«лучший способ — статичный контент сначала, динамичный в конце»*. По прайсингу Anthropic: *«Попадание в кеш стоит 10% от обычной input-цены»*. На практике это означает: стабильный системный контент (CLAUDE.md, skill деки, токены дизайна) → доки библиотеки компонентов → диалог. Не добавляй timestamp-ы и не меняй набор tools посреди сессии — оба ломают кеш.

**f) `/compact` на границах задач, `/clear` между несвязанными правками.** Из best practices Claude Code: *«Настраивай поведение compaction в CLAUDE.md инструкциями вроде 'при компактизации всегда сохраняй полный список изменённых файлов…'»*. Гарантия авто-компактизации задана в доке Claude Code Skills: *«Re-attached skills делят бюджет в 25 000 токенов»* — поэтому даже после компактизации твой deck-skill переживёт это, если его триггерный объём держится под 5000 токенов.

**g) Большие выводы — на диск, а не в контекст.** Когда просишь сабагента сканировать или сгенерировать что-то крупное (например, вытащить копирайт из брифа в PDF), скажи ему писать файл и возвращать путь + одна строка summary, а не полный текст.

**h) `frontend-design` skill от Anthropic — твой второй skill.** Поставь (`/plugin marketplace add anthropics/skills`) — он триггерится на «build web components, pages, artifacts, posters, or applications» и принуждает к anti-slop правилам. Из его SKILL.md: *«НИКОГДА не используй generic AI-эстетику: затасканные семейства шрифтов (Inter, Roboto, Arial, системные), клишированные цветовые схемы (особенно фиолетовые градиенты на белом), предсказуемые layouts и компонентные паттерны, штампованный дизайн без контекста»*. Сочетай с твоим deck-skill.

**i) MCP — только когда надо.** Эмпирическое правило сообщества (приписывается Шриву Шанкару): *«Если на MCP уходит больше 20K токенов — ты ломаешь Claude»*. Отключай Figma/database MCP, если они не нужны прямо сейчас.

### 4. Топ-5 проверенных практик (с источниками)

1. **Один HTML/Markdown-файл на слайд + один общий CSS-файл с переменными (PPTX-skill Anthropic).** SKILL.md: *«Создай HTML-файл на каждый слайд с правильными размерами»*; *«В общем .css-файле переопределяй CSS-переменные для цветов, типографики, отступов. НЕ создавай классы в этом файле»*.

2. **Используй `DISTILLED_AESTHETICS_PROMPT` дословно.** Cookbook Anthropic (Притви Раджасекаран, 21 октября 2025): скопируй блок `<frontend_aesthetics>…</frontend_aesthetics>` в `DESIGN.md` или в свой skill. Он явно банит: *«затасканные семейства шрифтов (Inter, Roboto, Arial, системные), клишированные цветовые схемы (особенно фиолетовые градиенты на белом), предсказуемые layouts и компонентные паттерны, штампованный дизайн без контекста»*.

3. **Дисциплина кеш-префикса (Тарик Шихипар / команда Claude Code, 30 апреля 2026).** *«Статика впереди, динамика в конце»*; не переставляй tools, не клади timestamp в системный промпт; *«prompt caching — это всё»*.

4. **Мульти-вход Slidev — ключевой примитив экономии токенов (Антоний Фу / доки Slidev).** `--- src: ./pages/intro.md ---` позволяет Claude читать или править отдельный файл на 50 токенов, а не всю деку. В сочетании с кастомными `layouts/` и `components/` исходник слайда становится почти чистым контентом.

5. **Визуальная валидация через сабагент + Playwright (skills Anthropic + zarazhangrui/frontend-slides).** Сделай скриншот 1920×1080 на каждый слайд, отдай сетку свежему сабагенту, спроси *«найди проблемы»* — правь в HTML/Markdown, повторяй. У официального PPTX-skill для этого есть `scripts/thumbnail.py`.

## Рекомендации (по этапам, с порогами)

**Этап 0 — на этой неделе (дёшево, сразу):**
1. Добавь `CLAUDE.md` (≤120 строк): build-команды, палитра + шрифты, *«правь `pages/*.md`, а не `slides.md`»*, ссылка на `DESIGN.md`.
2. Добавь `DESIGN.md` с дословным `DISTILLED_AESTHETICS_PROMPT` из Frontend Aesthetics Cookbook плюс твоей конкретной палитрой и тоном (например, «editorial / тёмный монохром с одним цитрусовым акцентом»).
3. Поставь плагин `frontend-design` от Anthropic (`/plugin marketplace add anthropics/skills`).
4. Настрой порядок, дружественный кешу: стабильный системный промпт + skill + tokens.css → диалог. Проверь это просто: после простого follow-up хода не должно быть свежей записи в кеш.

**Этап 1 — миграция одной деки (1–2 дня):**
1. Подними Slidev: `pnpm create slidev`. Перенеси текущие 12–20 HTML-слайдов в `pages/01-…md … pages/20-…md`.
2. Вытащи повторяющиеся визуальные паттерны в 5–8 кастомных Vue-компонентов в `components/` (например, `<DeviceMock>`, `<Stat>`, `<Quote>`, `<Chart>`).
3. Перенеси все цвета/шрифты/отступы в `styles/tokens.css` как CSS-переменные.
4. Растеризуй сложные мокапы устройств в PNG (или конвертируй в SVG); ссылайся из `public/`. (этот пункт под вопросом -- когда приступишь, надо будет посоветоваться со мной)
5. Напиши `.claude/skills/deck/SKILL.md`, документирующий контракт компонентов и правила правок.
6. Проверь, что `slidev export` даёт приемлемый PDF; если нужен PPTX с выделяемым текстом — добавь PPTX-skill Anthropic для финального шага.

**Пороги для смены курса:**
- Если после миграции исходник одного слайда стабильно превышает ~500 токенов — ты вытащил недостаточно в компоненты, рефакторь.
- Если один редактирующий ход тянет в контекст больше ~30K токенов файлов — CLAUDE.md/skills раздуты; разбей skill на reference-файлы, подгружаемые по требованию.
- Если cache hit rate плохой (видимые скачки потребления токенов после маленьких правок) — проверь: timestamp в CLAUDE.md, дёрганье набора MCP, переупорядочивание tools.
- Если `frontend-design` всё равно выдаёт generic — переключись с *«используй мой deck-skill»* на явный вызов (`/skill frontend-design`) и подсунь конкретную референс-деку или скриншот.
